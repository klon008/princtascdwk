## 1. Большая картина — что это за система

### Аналогия из реальной жизни

Представьте **альбом для наклеек** или **колоду Pokémon**:

- Есть **фиксированный набор** картинок (у нас — 27 персонажей).
- Вы **не покупаете карту напрямую** — вы покупаете **закрытый пакет (бустер)** и не знаете заранее, что внутри.
- Карты **разной редкости** — обычные выпадают часто, легендарные — редко.
- Собранный альбом можно **показать друзьям** — это и есть страница коллекции на сайте.

### Зачем это на стриме GoodGame

Стример ведёт трансляцию. Зрители смотрят, пишут в чат, участвуют в активности. За это они получают **баллы канала** (внутренняя валюта платформы GoodGame — как channel points на Twitch).

**Коллекционные карты** — это **долгая игра внутри канала**:

| Зачем | Пояснение |
|-------|-----------|
| Цель для зрителя | «Собрать альбом», «выбить Secret Rare», «догнать соседа по чату» |
| Куда тратить баллы | Бустеры — естественный «sink» (отток) накопленных баллов |
| Контент на стриме | Открытие бустера = маленькое событие в чате («О, Epic!») |
| Красивый результат | Галерея на сайте, ссылкой можно поделиться |

**Чем это НЕ является (на данном этапе):**
- Не магазин за реальные деньги
- Не pay-to-win в других играх бота
- Не торговая площадка между зрителями (об этом — [§9](#9-что-не-делаем-сейчас-торговля))

## 2. Участники и где что живёт

Система состоит из **трёх частей**, которые **не связаны напрямую** — только через правила и формат данных.

```
┌─────────────────────────────────────────────────────────────────┐
│  СТРИМ GoodGame                                                 │
│  Зрители пишут в чат: !бустер, !коллекция                       │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  PYTHON-БОТ (ваш ПК, локально)                                  │
│  • Читает чат GoodGame                                          │
│  • Хранит баллы, карты, бустеры в SQLite                        │
│  • Админка OBS: стример создаёт бустеры и шансы                 │
│  • При !коллекция — собирает ссылку с зашифрованными данными    │
└────────────────────────────┬────────────────────────────────────┘
                             │  ссылка ?c=...
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  САЙТ (GitHub Pages, в интернете)                               │
│  • Статическая страница, без своей базы данных                  │
│  • Расшифровывает ?c= и рисует красивую галерею               │
│  • Показывает ТОЛЬКО карты, которые уже у игрока               │
└─────────────────────────────────────────────────────────────────┘
```

### Почему сайт не «подключается» к боту

GitHub Pages — это **просто файлы в интернете** (HTML, JavaScript). Там **нет сервера**, который мог бы спросить ваш ПК: «какие карты у пользователя X?».

Поэтому бот **упаковывает снимок коллекции** в длинную строку в адресе ссылки:

```
https://....github.io/princtascdwk/?c=ЗАШИФРОВАННАЯ_СТРОКА
```

Сайт **сам расшифровывает** эту строку в браузере и показывает карты. База бота при этом **не нужна** сайту.

**Важно:** ссылка — **фотография на момент** команды `!коллекция`. Получил новую карту → нужно снова написать `!коллекция`, чтобы получить обновлённую ссылку.

## 3. Баллы канала — валюта стрима

**Баллы канала GoodGame** — это внутренняя валюта канала. Зрители их **зарабатывают** (смотрят стрим, пишут в чат, участвуют в активности) и **тратят** на награды канала.

В вашем боте баллы **уже учитываются** в SQLite (списки пользователей, обмены, социальные игры). Карты — **ещё один способ потратить баллы**, не замена существующей системе.

**Бустер стоит фиксированное число баллов.** Конкретную цену задаёт стример при создании бустера в админке (например, 1000 баллов за пак из 6 карт).

## 4. Что такое карта и бустeр

### Карта

Одна из **27** уникальных карт в текущей серии «Фантастический коллекционер». Каждая карта:

- **Персонаж** — Золушка, Ариэль, Королева Эльза и т.д.
- **Редкость** — от Common до Secret Rare (6 уровней).
- **Портрет и история** — отображаются на сайте при просмотре карты.

У игрока в коллекции карта **либо есть, либо нет**. Две одинаковые «Ариэль» в альбоме **не копятся** — при дубле срабатывает правило компенсации ([§6](#6-дубли--что-происходит-и-почему)).

### Бустер

**Бустер** — закрытый пакет карт за баллы. Стример **создаёт бустер в админке** бота (через OBS-панель) и настраивает:

- **Название** (например, «Тираж № 001»)
- **Цену** в баллах (например, 1000)
- **Сколько карт** выдаётся за одно открытие (например, 6)
- **Пул карт** — какие карты могут выпасть из этого бустера
- **Шансы редкости** — какой % Common, Uncommon, Rare и т.д.

Зритель покупает **целый бустер** — получает **несколько карт за раз** (не одну).

### Один активный бустер (РЕШЕНО)

В каждый момент времени **только один бустер активен** — аналогия с **системой баннеров в Genshin Impact**: идёт текущий «тираж», стример может подготовить следующий и **заменить** активный через админку, но **два бустера одновременно** зрители открыть не могут.

- Команда в чате: просто `!бустер` (без выбора id).
- Лимит открытий **в день на пользователя** — настраивается в админке OBS (если нужен).

### Промо-рисунок бустера (важно)

Игрокам **не показывают** на сайте коллекции «серые» или «заблокированные» карты, которых у них ещё нет. Иначе они увидят **весь список**, включая Secret Rare — это портит сюрприз и коллекционный азарт.

**Вместо этого:** для каждого бустера делается **прoмo-картинка (.jpg)** — красивый постер со **списком карт, которые могут выпасть из этого бустера**. Её показывают:

- на стриме (OBS),
- в описании канала,
- в чате бота при команде вроде `!бустер инфо`,
- где угодно **кроме** страницы «моя коллекция».

**Страница коллекции** — только **трофеи**: то, что игрок **уже выбил**.

## 5. Как зритель получает карту — пошагово

### Полный путь от нуля до галереи

**Шаг 1.** Зритель `@Player` копит баллы на канале GoodGame, смотря стрим.

**Шаг 2.** В чате пишет команду (точное имя команды настраивается в боте), например:
```
!бустer
```
(активен ровно один бустер — см. [§4](#4-что-такое-карта-и-бустer)).

**Шаг 3.** Бот проверяет:
- достаточно ли баллов;
- не нарушен ли кулдаун (если настроен);
- существует ли такой бустер и активен ли он.

**Шаг 4.** Бот **списывает** стоимость бустера (например, 1000 баллов).

**Шаг 5.** Бот **N раз** тянет случайную карту (N = число карт в бустере, например 6). Для каждой карты:
- выбирается редкость по шансам из админки;
- выбирается конкретная карта из пула бустера с этой редкостью;
- если карты **ещё нет** у игрока → **добавляется в коллекцию**;
- если карта **уже есть** (дубль) → **не добавляется**, начисляется компенсация ([§6](#6-дубли--что-происходит-и-почему)).

**Шаг 6.** Бот пишет в чат результат, например:
```
@Player открыл «Тираж № 001»:
  ✦ Ариэль (Rare) — новая!
  ✦ Золушка (Common) — новая!
  ✦ Белль (Uncommon) — дубль, возврат 42 балла
  ...
Итого: 4 новые карты, 84 балла возвращено. Коллекция: 12 карт. !коллекция — ссылка на альбом.
```

**Шаг 7.** Зритель пишет:
```
!коллекция
```

**Шаг 8.** Бот формирует ссылку с зашифрованным списком **только имеющихся** карт и отправляет в чат.

**Шаг 9.** Зритель открывает ссылку в браузере → видит галерею **своих** карт, заголовок с **ником**, может **скинуть ссылку другу**.

## 6. Дубли — что происходит и почему

### Что такое дубль

Игрок открывает бустер и выпадает карта, которая **уже есть** в его коллекции. Вторая такая же в альбом **не кладётся** — в альбоме каждый персонаж **в одном экземпляре**.

### Правило (РЕШЕНО)

При дубле игрок получает **25% стоимости одной карты** в этом бустере, **в баллах канала**.

**Формула:**

```
цена_одной_карты = стоимость_бустера / количество_карт_в_бустере
компенсация      = floor(цена_одной_карты × 0.25)   ← округление ВНИЗ
```

**Округление вниз (РЕШЕНО):** иначе при **полном паке дублей** суммарный возврат может **превысить** стоимость бустера. Пример: 6 дублей × 42 = 252 при cost 1000 — без floor риск выше.

**Пример:**

| Параметр | Значение |
|----------|----------|
| Стоимость бустера | 1000 баллов |
| Карт в бустере | 6 |
| Цена одной «позиции» | 1000 ÷ 6 ≈ **167** баллов |
| Компенсация за дубль | 167 × 0.25 ≈ **42** балла |

Если в одном бустере **два дубля** — компенсация **дважды** (2 × 42 = 84).

### Почему именно так

- Игрок **не чувствует**, что дубль = полный ноль (часть баллов вернулась).
- Не нужна вторая валюта («пыль», «осколки») — проще для понимания.
- Не нужна торговля «лишними» картами — дубль сразу конвертируется в баллы.
- Стример контролирует экономику через **цену бустера** и **число карт в паке**.

### Что пишет бот при дубле

```
✦ Белль (Uncommon) — уже в коллекции. Возврат: 42 балла.
```

Карта **не появляется** повторно на сайте коллекции.

## 7. Редкость и шансы выпадения

### Шесть уровней редкости

| Редкость | Роль в игре | Примеры в сете |
|----------|-------------|----------------|
| **Common** | Частые, база альбома | Золушка, Олаф, Паскаль… |
| **Uncommon** | Чуть приятнее | Белль, Жасмин… |
| **Rare** | Заметная удача | Ариэль, Моана… |
| **Epic** | Редкость, повод написать в чат | Рапунцель, Рая… |
| **Legendary** | Большой момент | Аврора |
| **Secret Rare** | Главная мечта; в сете их **2** | Королева Эльза, Мегара |

### Кто задаёт проценты (РЕШЕНО)

**Не** зашиваются в код жёстко. У бота есть **админка для стримера** (OBS-панель): функционал **«Создание бустеров»**, где стример сам указывает:

- состав пула;
- веса / проценты по редкостям;
- цену и число карт.

Когда стример **меняет** активный бустер — у нового тиража свои шансы и свой promo JPG.

### Pity / гарантии редкости

**Не входит в MVP.** Архитектуру под pity **сейчас не закладываем.**

Стример уже **задумывался** о механике вроде «каждые N бустеров без Rare — гарант Rare+». Теоретически это **возможно в будущем** (отдельная доработка админки и логики roll), но на текущем этапе — **вне scope**.

## 8. Команда !коллекция и сайт

### Что делает команда

```
Зритель: !коллекция

Бот: Коллекция @Player (12 из 27): https://....github.io/princtascdwk/?c=длинная_строка
```

### Что видит человек на сайте (РЕШЕНО)

| Элемент | Показываем? |
|---------|-------------|
| **Ник владельца** | **Да, обязательно** — крупно: «Коллекция @Player» |
| **Счётчик прогресса** | **Да — «12 из 27»** (и в чате бота, и на сайте) |
| **Только полученные карты** | **Да** — сетка из карт, которые есть в коллекции |
| **Неполученные / «замочки» / силуэты** | **Нет, никогда** — не показывать *каких именно* карт не хватает |
| **Дата и бустер на карточке** | Да — когда и из какого тиража получена |
| **«Первым вытащил @Other»** | **Нет** на данном этапе |
| **Поделиться ссылкой с другом** | **Да** — ссылка публичная, кто имеет URL, тот видит коллекцию |

**Зачем «12 из 27», но без списка отсутсвующих:** игрок **видит незавершённость** (дискомфорт «дело не доделано»), но **не знает**, каких именно 15 карт нет — мотивация копить и открывать бустеры, без спойлера Secret Rare на странице коллекции.

### Страница без ссылки (?c= отсутствует)

Если человек зашёл на сайт **напрямую**, без параметра `?c=`:

- Краткое объяснение: «Получите ссылку командой !коллекция в чате стрима».
- **Не** показывать полный каталог карт.
- Можно показать **общий промо-арт** серии (не список всех 27 с редкостями).

### Где смотреть «что может выпасть»

На **promo .jpg** конкретного бустера (см. [§4](#4-что-такое-карта-и-бустeр)), не на странице коллекции.

## 9. Что НЕ делаем сейчас (торговля)

### Решение (РЕШЕНО)

**Обмен картами и выкуп друг у друга — не делаем.** Возможно в **далёком будущем**; сейчас архитектура **без торговли**.

### Что это значит practically

| Не делаем сейчас | Делаем вместо этого |
|------------------|---------------------|
| `!обмен`, `!купить`, маркетплейс | Простая модель: одна карта типа на аккаунт |
| Уникальные `instance_id` для трейда | Запись `(user_id, card_id)` в SQLite |
| «Кто первый вытащил карту» | Не храним и не показываем |
| История сделок между игроками | Только лог открытий бустеров (аудит) |

Если торговлю **когда-нибудь** добавят — потребуется **рефакторинг** (миграция БД, новые команды, другой payload в URL). Это осознанный компромiss ради простоты MVP.

## 10. Связь с остальным ботом

Карты — **дополнительная** механика, не замена существующим функциям:

```
Баллы канала (SQLite бота)
    ├── социальные игры (уже есть)
    ├── обмены баллами между игроками (уже есть)
    └── бустеры → коллекция карт → ссылка на сайт
```

**Карты влияют на другие игры бота?** **Нет** (на данном этапе) — чистая коллекция, без бонусов в соц. играх.

## 11. Решения — зафиксировано

| # | Тема | Решение |
|---|------|---------|
| 1 | Связь бот ↔ сайт | Зашифрованный параметр `?c=` в URL, без синхронизации с GitHub |
| 2 | Торговля | **Не делаем**; архитектура простая; рефакторинг потом при необходимости |
| 3 | Дубли | **25%** от `(стоимость_бустера / число_карт)` баллов за каждый дубль; карта не добавляется |
| 4 | Шансы редкости | Стример задаёт в **админке создания бустеров** (OBS) |
| 5 | Сайт коллекции | **Только полученные** карты; счётчик **«X из 27»**; без силуэтов отсутсвующих |
| 6 | Состав бустера | **Promo JPG**, не сайт коллекции |
| 7 | Ник на странице | **Обязательно** |
| 8 | Ссылка другу | **Да**, публичная |
| 9 | «Первым вытащил» | **Нет** |
| 10 | Модель инвентаря | Одна карта каждого типа на пользователя `(user_id, card_id)` |
| 11 | Счётчик коллекции | **«12 из 27»** — прогресс виден, список отсутсвующих — нет |
| 12 | Активные бустеры | **Один** одновременно (как баннер Genshin) |
| 13 | Лимит бустеров в день | Настройка в **админке OBS** |
| 14 | Округление дубля | **Вниз** (`floor`) — защита от переплаты при полном паке дублей |
| 15 | Pity / гарантии | **Не MVP**, архитектуру не закладываем; теоретически возможно позже |

## 13. Сценарии для проверки логики

### Сценарий A — новичок

`@Newbie` впервые на стриме, 1000 баллов, открывает бустер (6 карт).

- Списалось 1000 баллов.
- Выпало 6 карт, все новые → 6 записей в коллекции.
- `!коллекция` → ссылка → на сайте **«6 из 27»**, ник `@Newbie`, сетка из 6 карт.
- **Не видно**, *каких именно* 21 карты не хватает — только число.

### Сценарий B — дубли в одном паке

Бустер 1000 / 6 карт. Два дубля.

- Компensation: 2 × 42 = **84** балла возвращено.
- Новые карты добавлены, дубли — нет.
- Бот явно пишет про каждый дубль и возврат.

### Сценарий C — друг открыл чужую ссылку

`@Friend` получил URL от `@Player`.

- Видит коллекцию **@Player** (ник на странице).
- Видит **«12 из 27»** и **только** карты @Player.
- Не видит, *каких именно* карт не хватает — только общий прогресс.

### Сценарий D — полный альбом по типам

У `@Veteran` уже все 27 карт. Открывает ещё один бустер.

- Все 6 выпавших — дубли.
- Все 6 → компенсация 25%, **ничего нового** в коллекции.
- Ссылка `!коллекция` показывает **«27 из 27»** — полный альбом, без новых карт.

### Сценарий E — стример сменил бустер (новый тираж)

В админке: **деактивирован** старый бустер, **активирован** новый — другой пул, шансы, promo JPG.

- `!бустер` тянет только из **текущего** активного тиража (как смена баннера).
- Promo JPG нового тиража — в инфо-команде / на стриме.
- Коллекция игрока **накапливается** между тиражами; счётчик по-прежнему **X из 27**.
