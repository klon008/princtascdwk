## Краткий обзор

**Для игрока.** Зритель копит баллы канала на стриме GoodGame и тратит их на открытие бустера командой `!бустер`. Выпавшие карты попадают в личный альбом. Командой `!альбом` игрок получает **короткую** ссылку на сайт (GitHub Pages); страница запрашивает актуальный альбом с API бота через туннель [CloudPub](https://cloudpub.ru/) на ПК стримера.

**Для стримера.** В админке OBS создаются бустеры (пулы карт) и тиражи (цена, число карт, шансы). В каждый момент активен один тираж — аналог баннера в Genshin Impact. На время стрима поднимается локальный API и туннель CloudPub; вне эфира просмотр альбома может быть недоступен — это допустимо на данном этапе.

**Для художника.** Карты объединяются в серии внутри общей Коллекции. Для каждого бустера готовится promo JPG с перечнем карт, которые могут выпасть. Состав пула на альбоме игрока не дублируется.

## 1. Большая картина



### Аналогия из реальной жизни

Представьте **коллекционный альбом наклеек** ([Panini](https://www.iz90.ru/blog/statya-198) и подобные):

- Есть **вся Коллекция** — полный список всех наклеек в игре.
- Альбом разбит на **страницы** — каждая страница соответствует серии.
- Наклейку нельзя купить напрямую — покупается **пакет (бустер)**, содержимое заранее неизвестно.
- Собранные наклейки отображаются в **личном альбоме** на сайте; ссылкой можно поделиться.



### Уровни достижений


| Уровень                     | Смысл                                                      |
| --------------------------- | ---------------------------------------------------------- |
| Собрать **серию**           | Все карты одной «страницы» альбома                         |
| Собрать **несколько серий** | Несколько страниц заполнены полностью                      |
| Собрать всю **Коллекцию**   | Каждая карта из полного списка системы — высшее достижение |




### Зачем это на стриме GoodGame

Стример ведёт трансляцию. Зрители смотрят, пишут в чат, участвуют в активности и получают **баллы канала** (внутренняя валюта GoodGame).

Коллекционные карты создают долгосрочную игровую цель внутри канала: зрители не просто копят баллы, а стремятся собрать серию или всю Коллекцию.


| Цель                           | Как реализуется                                                   |
| ------------------------------ | ----------------------------------------------------------------- |
| Дать зрителю долгосрочную цель | Собрать серию, затем всю Коллекцию, выбить Mythic / Secret Rare   |
| Создать расход для баллов      | Открытие бустеров — способ потратить накопленные баллы            |
| Добавить контент в чат         | Каждое открытие — событие; выпадение редких карт вызывает реакцию |
| Показать прогресс наглядно     | Альбом на сайте с фильтрами и счётчиками прогресса                |


**Вне scope на данном этапе:**

- Магазин за реальные деньги
- Pay-to-win в других играх бота
- Торговая площадка между зрителями ([§11](#11-ограничения-mvp-обмен-между-игроками))

**MVP:** одна серия «Фантастический коллекционер» (28 карт + рубашка серии). Архитектура спроектирована так, чтобы поддерживать много серий и большую Коллекцию в будущем.

## 2. Глоссарий

```
Коллекция ────── все возможные карты в системе
    └── Серия ── тематический поднабор («страница» альбома)
            └── Карта

Бустер ───────── пул карт для открытия
    └── Тираж ── правила открытия: цена, N карт, шансы

Альбом ───────── карты, которые уже есть у игрока (на сайте)
Открытие ─────── действие в чате: !бустер
```


| Термин        | Определение                                                                                  | Аналогия                                                          |
| ------------- | -------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| **Коллекция** | Полный список всех карт в системе. Собрать всю Коллекцию — высшее достижение                 | Весь [альбом Panini](https://www.iz90.ru/blog/statya-198) целиком |
| **Серия**     | Тематический поднабор внутри Коллекции. Каждая карта принадлежит ровно одной серии. У серии есть **рубашка** — единый дизайн оборота карт этой серии | Страница в альбоме наклеек                                        |
| **Бустер**    | Конфигурация пула: какие карты могут выпасть при открытии. Не привязан один к одному к серии | Пул карт в booster pack                                           |
| **Тираж**     | Конфигурация правил открытия бустера: цена, число карт, шансы. Не изменяется после создания  | Баннер в Genshin Impact                                           |
| **Альбом**    | Набор карт, которые игрок уже получил; отображается на сайте                                 | Личная наклеенная часть альбома                                   |
| **Открытие**  | Покупка и разбор пака за баллы: команда `!бустер` в чате                                     | Открытие пака / бросок                                            |




### Как не путать термины


| Контекст              | Как говорить                                     |
| --------------------- | ------------------------------------------------ |
| Все карты в игре      | «В **Коллекции** 120 карт в 8 сериях»            |
| То, что у игрока      | «В твоём **альбоме** 12 карт»                    |
| Команда в чате        | `!альбом` → ссылка на **альбом** игрока на сайте |
| Прогресс по странице  | «Серия „Принцессы“: **8 из 15**»                 |
| Прогресс по всей игре | «**Коллекция**: 45 из 120»                       |




### Связи между сущностями

- **Серия** и **бустер** — разные сущности. Бустер выбирает карты из Коллекции; в одном бустере могут быть карты из нескольких серий.
- **Одна карта** — только в **одной серии**, но может входить в **разные бустеры**.
- **Тираж** всегда принадлежит **одному бустеру**. У бустера может быть много тиражей за историю.
- **Открытие** всегда идёт по правилам **одного активного тиража**.

## 3. Участники и где что живёт

```
┌─────────────────────────────────────────────────────────────────┐
│  СТРИМ GoodGame                                                 │
│  Зрители: !бустер (открытие), !альбом (короткая ссылка)         │
│  Лимит сообщения в чате: ~4 КБ (~3000–3500 символов)            │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  PYTHON-БОТ (локально, SQLite)                                  │
│  • Коллекция, серии, бустеры, тиражи, альбомы игроков           │
│  • Админка OBS: localhost (отдельный порт, не в туннеле)        │
│  • HTTP API: localhost:8080 — только чтение альбома             │
│  • !альбом → короткая ссылка ?u=ник&k=подпись                   │
└────────────┬───────────────────────────────┬────────────────────┘
             │                               │
             │ на время стрима               │
             ▼                               ▼
┌────────────────────────────┐   ┌──────────────────────────────────┐
│  CloudPub (туннель)        │   │  САЙТ (GitHub Pages)             │
│  api.домен → :8080 на ПК   │   │  React UI, арты карт (статика)   │
│  HTTPS, свой домен         │   │  fetch(JSON) → API через туннель │
└────────────────────────────┘   └──────────────────────────────────┘
```

**Разделение ответственности:**


| Где                  | Что хранится / делается                                                           |
| -------------------- | --------------------------------------------------------------------------------- |
| ПК стримера (SQLite) | Источник правды: инвентарь, тиражи, каталог карт                                  |
| CloudPub             | Публичный HTTPS-доступ к локальному API **без** света домашнего IP                |
| GitHub Pages         | Оболочка UI и статические арты; **без** секретов и полного каталога в репозитории |
| Чат GoodGame         | Короткая ссылка на сайт; данные альбома **не** передаются в URL                   |


**Ссылка из** `!альбом`**:**

```
https://....github.io/princtascdwk/?u=player&k=SIGNED_TOKEN
```

Сайт по `u` и `k` запрашивает `GET /api/v1/album` у API через CloudPub. Альбом **живой**: после новой карты достаточно обновить страницу (повторный `!альбом` не обязателен, но допустим для новой подписи).

**Доступность:** туннель и API работают, пока идёт стрим (или пока стример держит их включёнными). Вне эфира зритель может увидеть сообщение «альбом доступен на стриме» — на MVP это допустимо ([§15](#15-запасные-планы) — запасные варианты).

**Безопасность:** админка OBS и служебные порты **не** публикуются через CloudPub — только read-only API альбома.

## 4. Баллы канала

**Баллы канала GoodGame** — внутренняя валюта канала. Зрители зарабатывают их активностью и тратят на награды.

В боте баллы уже учитываются (социальные игры, обмены). Карты — дополнительный способ их потратить.

**Цена открытия** задаётся в тираже (например, 1000 баллов за 6 карт).

## 5. Коллекция, серии и карты



### Коллекция

**Коллекция** — все карты, существующие в системе: мастер-список с id, именем, редкостью, артом, привязкой к серии.

- Растёт со временем: художник добавляет новые серии и карты.
- **Собрать всю Коллекцию** — главная долгосрочная цель игрока.



### Серия

**Серия** — тематический поднабор внутри Коллекции. Формируется художником: стиль, тема, время создания. Аналогия — **страница** в коллекционном альбоме.

**Правила:**

- Каждая карта принадлежит ровно одной серии.
- Разные серии не содержат одинаковых карт.
- Прогресс в серии: «**8 из 15**» — заполненность страницы.
- У каждой серии есть **рубашка** (card back) — визуальная характеристика серии: единый дизайн оборота карт. Рубашка не является коллекционной картой и не входит в счётчик «N из M»; на сайте показывается как декоративный элемент каталога серии.

**MVP:** серия «Фантастический коллекционер» — 28 карт + рубашка серии.

### Карта

У каждой карты: персонаж, редкость (Common → Uncommon → Rare → Epic → Legendary → Mythic → Secret Rare), портрет, история.

В альбоме игрока карта либо есть, либо нет (один экземпляр). При дубле — компенсация баллами ([§8](#8-дубли-и-компенсация)).

## 6. Бустер и тираж



### Бустер

**Бустер** — конфигурация **пула карт**: из каких карт (возможно из разных серий) могут выпасть награды при открытии.

Стример создаёт бустер в админке OBS:

- название («Сундук классики», «Стартовый набор»);
- список карт в пуле;
- promo .jpg — постер с картами, которые могут выпасть (не на странице альбома).

Бустер — это пул карт, из которого случайным образом выпадают награды. При открытии бот делает **N независимых бросков**: каждый раз отдельно определяется редкость и выбирается карта из пула бустера.

Бустер и серия не связаны напрямую: бустер «Микс» может включать карты из разных серий.

### Тираж

**Тираж** — конфигурация **правил открытия** для конкретного бустера:


| Параметр              | Пример                  |
| --------------------- | ----------------------- |
| Цена в баллах         | 1000                    |
| Карт за одно открытие | 6                       |
| Шансы по редкостям    | Common 50%, Rare 10%, … |
| Привязка к бустеру    | Бустер № 15             |


Аналогия: **баннер в Genshin Impact** — активен один, у него свои ставки и цена; пул карт задан бустером.

У одного бустера — много тиражей за историю. Примеры ротации:


| Период   | Активный тираж | Бустер    |
| -------- | -------------- | --------- |
| Неделя 1 | Тираж 1        | Бустер 15 |
| Неделя 2 | Тираж 40       | Бустер 1  |
| Неделя 3 | Тираж 2        | Бустер 15 |


В чате всегда `!бустер` — открывается текущий активный тираж (без выбора id).

### Правила тиража


| Действие                                                  | Статус            | Комментарий                                 |
| --------------------------------------------------------- | ----------------- | ------------------------------------------- |
| Создать новый тираж                                       | Поддерживается    | —                                           |
| Сделать тираж активным (остальные становятся неактивными) | Поддерживается    | В каждый момент активен только один тираж   |
| Приостановить активный тираж                              | Поддерживается    | Открытия недоступны, пока тираж на паузе    |
| Редактировать шансы / цену / пул у существующего тиража   | Не поддерживается | Для изменения правил создаётся новый тираж  |
| Удалить тираж из истории                                  | Не поддерживается | Тираж остаётся в журнале                    |
| Перезапустить тот же тираж                                | Не поддерживается | Только копия настроек → новый уникальный id |


Чтобы изменить шансы: создать новый тираж (можно скопировать настройки старого) и активировать его. Два тиража с одним пулом бустера, но разными шансами — два разных тиража одного бустера.

### Один активный тираж

В каждый момент ровно один тираж в статусе `active`. Зрители открывают только его.

Лимит открытий в день на пользователя — настройка в админке OBS.

### Promo JPG

Состав пула бустера показывают на promo .jpg (стрим, `!бустер инфо`) — не на странице альбома.

## 7. Как игрок получает карту

**Шаг 1.** Игрок копит баллы на канале.

**Шаг 2.** Пишет в чат:

```
!бустер
```

**Шаг 3.** Бот проверяет: достаточно ли баллов, лимит в день, кулдаун (если настроен), активен ли тираж (не на паузе).

**Шаг 4.** Списывает цену тиража (например, 1000 баллов).

**Шаг 5.** N раз (N из тиража, например 6):

1. Бросок редкости по весам тиража.
2. Случайный выбор карты из пула бустера с этой редкостью.
3. Карты нет в альбоме → добавить; дубль → компенсация ([§8](#8-дубли-и-компенсация)).

**Шаг 6.** Бот пишет в чат, например:

```
@Player · Тираж № 40 · Бустер «Старт»
  ✦ Ариэль (Rare) — новая!
  ✦ Золушка (Common) — новая!
  ✦ Белль (Uncommon) — дубль, возврат 42 балла
  ...
Итого: 4 новые. Альбом: 12 карт. Серия «Фантастический коллекционер»: 12 из 28.
!альбом — ссылка на альбом.
```

**Шаг 7–9.** `!альбом` → короткая ссылка `?u=&k=` → сайт запрашивает API через CloudPub → альбом с ником, фильтрами по сериям, только полученные карты.

## 8. Дубли и компенсация

**Дубль** — выпала карта, которая уже есть в альбоме. Вторая копия не добавляется.

### Формула компенсации за один дубль

```
цена_позиции = стоимость_тиража / количество_карт_в_открытии
возврат      = floor(цена_позиции × 0.25)
```

Компенсация за дубли рассчитывается так, чтобы суммарный возврат при полном паке дублей не превышал стоимость открытия. Это достигается округлением вниз (`floor`).

**Пример:** тираж 1000 баллов, 6 карт → цена позиции ≈ 167 → возврат **42** балла за один дубль. Два дубля → 2 × 42 = 84 балла.

**Сообщение бота:**

```
✦ Белль (Uncommon) — уже в альбоме. Возврат: 42 балла.
```

## 9. Редкость и шансы

Семь уровней: Common → Uncommon → Rare → Epic → Legendary → Mythic → Secret Rare.

| Редкость     | Ключ в коде   | Роль                                      |
| ------------ | ------------- | ----------------------------------------- |
| Common       | `common`      | База                                      |
| Uncommon     | `uncommon`    | Чуть богаче                              |
| Rare         | `rare`        | Премиум-рамка                             |
| Epic         | `epic`        | Роскошь, лучи, филигрань                  |
| Legendary    | `legendary`   | Корона, ювелирная отделка                 |
| Mythic       | `mythic`      | Между Legendary и Secret Rare; magmatic FX |
| Secret Rare  | `secretRare`  | Эксклюзив, голография                     |

Шансы задаёт стример при **создании тиража** в админке OBS — не в коде.

### Гарантии редкости (pity)

Не входят в MVP. Архитектура под pity не закладывается.

Теоретически возможно в будущем (например, «каждые N открытий без Rare — гарант Rare+»). Отдельная доработка, когда понадобится.

## 10. Команда !альбом



### Команда

```
Зритель: !альбом

Бот: Альбом @Player · серия «Фантастический коллекционер»: 12 из 28 · Коллекция: 12 из 28
     https://....github.io/princtascdwk/?u=player&k=...
```

Сообщение должно укладываться в **лимит чата GoodGame (~4 КБ, ~3000–3500 символов)** — поэтому в URL только ник и короткая подпись, без передачи всего альбома.

При нескольких сериях в сообщении указывается прогресс по выбранной серии и общий по Коллекции.

### Альбом на сайте


| Элемент                  | Реализация                                                 |
| ------------------------ | ---------------------------------------------------------- |
| Ник владельца            | Отображается обязательно                                   |
| Общий альбом             | Все карты игрока в одном месте                             |
| Фильтры по сериям        | Быстрое переключение «страниц»                             |
| Прогресс по серии        | «8 из 15» для выбранной серии                              |
| Прогресс по Коллекции    | «45 из 120» — общая цель                                   |
| Полученные карты         | Сетка только из карт, которые есть у игрока                |
| Отсутствующие карты      | Силуэты и «замки» не показываются (см. пояснение ниже)     |
| Дата и тираж на карточке | Отображаются                                               |
| «Первым вытащил»         | Не отображается                                            |
| Ссылка другу             | Публичная: кто имеет URL, видит альбом (пока API доступен) |
| API недоступен           | Сообщение: альбом на стриме / туннель выключен             |


Не показывать силуэты или «замки» для отсутствующих карт. Игрок видит только полученные карты. Это сохраняет интригу: он знает, сколько карт не хватает (счётчик «X из Y»), но не знает, каких именно.

### Страница без параметров альбома

- «Получите ссылку командой !альбом в чате стрима».
- Полная Коллекция не показывается.
- Допустим общий промо-арт проекта.



### Где смотреть «что может выпасть»

Promo JPG активного бустера или команда `!бустер инфо` — не на альбоме.

## 11. Ограничения MVP: обмен между игроками

В MVP реализуем **простую модель инвентаря**: одна карта каждого типа на аккаунт, запись `(user_id, card_id)` в SQLite, лог открытий тиражей.

**Отложено:** обмен картами и выкуп между зрителями (`!обмен`, маркетплейс), уникальные `instance_id`, история сделок. При необходимости — рефакторинг в будущем.

## 12. Связь с остальным ботом

```
Баллы канала
    ├── социальные игры
    ├── обмены баллами
    └── тираж → открытие → альбом → ссылка на сайт
```

Карты **не влияют** на другие игры бота — чистая коллекция, без бонусов в социальных механиках.

## 14. Сценарии для проверки логики



### A — новичок, первая серия (MVP)

**Условие:** у игрока 1000 баллов; активен Тираж 40, Бустер «Старт», 6 карт за открытие; все 6 выпавших карт новые.

**Ожидаемый результат:** в альбоме 6 карт; `!альбом` ведёт на страницу с прогрессом «6 из 28» по серии.

**Детали:** сетка из 6 карт; фильтр по серии работает; не отображается, каких именно 22 карты не хватает — только счётчик.

### B — дубли

**Условие:** игрок открывает бустер (тираж 1000 баллов, 6 карт). Два выпавших предмета — дубли.

**Ожидаемый результат:** система начисляет возврат 2 × 42 = 84 балла; альбом не изменяется.

**Детали:** в чате — список выпавших карт с пометкой «дубль» и суммой возврата за каждую.

### C — другой пользователь открыл ссылку

**Условие:** один игрок передал URL от `!альбом` другому пользователю.

**Ожидаемый результат:** отображается альбом владельца ссылки с его ником и прогрессом.

**Детали:** видны только карты владельца; публичный доступ по URL; отсутствующие карты не показаны.

### D — серия собрана, Коллекция нет

**Условие:** игрок собрал все 28 карт серии «Фантастический коллекционер»; в Коллекции появилась новая серия из 15 карт.

**Ожидаемый результат:** по первой серии «28 из 28»; по Коллекции «28 из 43» — главная цель не закрыта.

**Детали:** в фильтре второй серии «0 из 15»; альбом накапливает карты сквозь серии.

### E — полный пак дублей

**Условие:** все 6 карт в открытии — дубли; в альбоме уже есть все выпавшие типы.

**Ожидаемый результат:** только возврат баллов; альбом без изменений.

**Детали:** суммарный возврат не превышает стоимость открытия (благодаря `floor`).

### F — ротация тиража между бустерами

**Условие:** неделя 1 — Тираж 1 / Бустер 15; неделя 2 — Тираж 40 / Бустер 1.

**Ожидаемый результат:** `!бустер` всегда использует активный тираж; альбом игрока накапливается сквозь смены.

**Детали:** карты из разных тиражей и бустеров попадают в один альбом; прогресс по сериям и Коллекции сохраняется.

### G — новый тираж вместо правки шансов

**Условие:** стример хочет удвоить шанс Epic; создаёт Тираж 41 (копия Тиража 40 с новыми весами) и активирует его.

**Ожидаемый результат:** открытия идут по Тиражу 41; Тираж 40 переведён в неактивный, не удалён.

**Детали:** история тиражей сохраняется; перезапуск того же id невозможен.

### H — API недоступен (туннель выключен)

**Условие:** зритель открыл ссылку от `!альбом`, пока стрим не идёт или CloudPub не поднят.

**Ожидаемый результат:** сайт показывает понятное сообщение («альбом доступен на стриме»), без падения и без утечки чужих данных.**CloudPub** 

**Детали:** ошибка сети / 503 обрабатывается на фронте; повторная попытка после начала эфира.

## 15. Запасные планы

Основной путь — [§3](#3-участники-и-где-что-живёт): **CloudPub + локальный API**. Если туннель не удаётся внедрить или сервис недоступен:

### План B — зашифрованный `?c=` в URL


|            |                                                                                                                          |
| ---------- | ------------------------------------------------------------------------------------------------------------------------ |
| **Суть**   | Весь альбом в зашифрованном query-параметре; сайт расшифровывает на клиенте                                              |
| **Плюсы**  | Не нужен туннель; альбом открывается без ПК стримера                                                                     |
| **Минусы** | Секрет в бандле сайта; полный каталог в репозитории; **лимит чата GoodGame ~4 КБ**; рост Коллекции упирается в длину URL |
| **Когда**  | Временный fallback на 1 серию / мало карт; не целевая архитектура                                                        |




### План C — виртуальный хостинг (API + БД на сервере)


|            |                                                                                      |
| ---------- | ------------------------------------------------------------------------------------ |
| **Суть**   | Бот синхронизирует альбомы на VPS/хостинг; сайт читает публичный API 24/7            |
| **Плюсы**  | Альбом всегда онлайн; нет зависимости от ПК и туннеля; масштаб                       |
| **Минусы** | Стоимость (в РФ выросла); синхронизация бот ↔ сервер; админка и БД на двух площадках |
| **Когда**  | Нужен альбом вне стрима; торговля; отказ от локальной модели                         |




### Порядок попыток

1. **CloudPub** (MVP) — см. [TODO_tech.md §2](./TODO_tech.md#2-архитектура-связи-бот--сайт)
2. При блокере — оценить **План B** только для узкого MVP
3. При росте нагрузки или требовании 24/7 — **План C**
